# Security Policy

## Supported Versions

Only the latest development build is actively maintained and receives security fixes.

| Version / Channel | Supported |
| ----------------- | --------- |
| Latest dev build (GitHub Actions) | ✅ |
| Older builds | ❌ |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

To report a vulnerability, contact us privately via one of the following:

- **Discord**: Open a private message or use the `#security` channel in our [Discord server](https://discord.gg/VHdwQFsaGX)
- **GitHub**: Use [GitHub's private vulnerability reporting](https://github.com/HyperLauncher/HyperLauncher/security/advisories/new)

### What to include

- A clear description of the vulnerability
- Steps to reproduce it
- The potential impact or attack scenario
- Your suggested fix (optional but appreciated)

### What to expect

- **Acknowledgement**: within 48–72 hours
- **Status update**: within 7 days of your report
- **Fix timeline**: depends on severity — critical issues are prioritized immediately

If a vulnerability is accepted, we will credit you in the release notes (unless you prefer to remain anonymous). If declined, we will explain why.

We appreciate responsible disclosure and thank you for helping keep Hyper Launcher safe.
